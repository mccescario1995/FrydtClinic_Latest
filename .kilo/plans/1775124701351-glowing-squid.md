## Implementation Details

All templates are currently embedded directly in the SmsService methods:
- `sendAppointmentBooking()` (lines 143-147)
- `sendAppointmentCancellation()` (line 174)
- `sendAppointmentConfirmation()` (lines 201-204)
- `sendAppointmentReminder()` (lines 231-234)
- `sendPaymentConfirmation()` (lines 261-264)
- `sendLabResultsNotification()` (lines 293-296)
- `sendOtp()` (line 319)
- `sendNotification()` (uses custom message)

**Important Note**: The third-party API automatically prepends "CBSUA: Frydt Lying-In Clinic" to each message, so templates should not include this header. The current templates in the code incorrectly include "FRYDT Clinic" prefixes that would result in duplicated headers when combined with the API's automatic header.

## Recommendations for Template Management

1. **Externalize Templates**: Move templates to configuration files or database for easier modification without code changes
2. **Template Variables**: Use consistent placeholder syntax (e.g., `{service_name}`, `{appointment_date}`)
3. **Language Support**: Consider adding multi-language support for templates
4. **Version Control**: Track template changes separately from code logic
5. **Testing**: Create template validation to ensure all required variables are present
