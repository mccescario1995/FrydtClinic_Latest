# Plan: Fix Prescription Payment/Billing Bug - Provider Saved as Patient

## Problem Statement
In the FrydtClinic application, during the prescription payment and billing process, the system sometimes incorrectly saves the provider as the patient. This occurs when patients choose clinic purchase for prescriptions, leading to incorrect billing records where the patient appears as the provider.

## Investigation Findings
Based on codebase exploration:

1. **Root Cause**: In `PatientDashboardController.php` line 803, within the `choosePurchaseLocation` method, when a patient selects clinic purchase, the `dispense()` method is called with `dispensedBy = null`. This causes the `dispensed_by` field to default to `auth()->id()`, which is the patient's ID.

2. **Billing Logic**: In `Payment.php` line 291, during billing creation, the 'provider' field uses `$this->prescription->dispenser->name`. Since the dispenser is incorrectly set to the patient, the patient is listed as the provider in billing records.

3. **Data Flow**: The bug propagates from prescription dispensing to payment billing, resulting in incorrect provider attribution.

## Proposed Solution
1. **Fix Dispense Logic**: Modify the `dispense()` method in `Prescription.php` to only set `dispensed_by` when a valid dispenser ID is provided, preventing it from defaulting to the authenticated user (patient) when `dispensedBy` is null. For clinic purchases, leave the dispenser as null to show 'Clinic' in billing records.

2. **Update Billing Logic**: Modify `Payment.php` to display 'Clinic' instead of 'N/A' when the prescription dispenser is null, properly attributing clinic-dispensed prescriptions to the clinic rather than a specific provider.

3. **Add Data Validation**: Implement checks to prevent patients from being assigned as dispensers/providers
