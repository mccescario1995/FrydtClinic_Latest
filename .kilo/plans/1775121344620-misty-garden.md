# Plan: Update Time Selection in Book Appointment Form

## Objective
Modify the appointment booking form to show all time slots but disable (gray out) times that are already booked, while still allowing users to see them.

## Current Implementation
- The time dropdown (`#appointment_time`) is populated via AJAX call to `/patient/api/available-time-slots`
- Only available time slots are returned and shown in the dropdown
- Unavailable times are completely hidden from the user

## Required Changes
1. **Backend Modification**: Update `getAvailableTimeSlots` method in `PatientDashboardController.php` to return both available and unavailable time slots with their status
2. **Frontend Modification**: Update the `loadTimeSlots` JavaScript function to:
   - Create enabled options for available times
   - Create disabled options for unavailable times (grayed out but visible)
   - Maintain the current behavior when no slots are available

## Files Modified
1. `app/Http/Controllers/PatientDashboardController.php` - Updated `getAvailableTimeSlots` method
2. `resources/views/patient/book-appointment.blade.php` - Updated `loadTimeSlots` JavaScript function

## Implementation Details

### Backend Changes
In `getAvailableTimeSlots`:
- Instead of returning only available slots, generate all possible slots based on schedule
- For each slot, determine if it's available (no overlap with existing appointments)
- Return array of objects with: `{ time, display, available: boolean }`

### Frontend Changes
In `loadTimeSlots` function:
- Parse the returned data to check the `available` property
- For available slots: create normal `<option>` elements
- For unavailable slots: create `<option>` elements with `disabled` attribute and apply CSS styling for grayed-out appearance
- Keep existing logic for handling empty results and error cases

## Testing Considerations
- Verify that available times remain selectable
- Verify that unavailable times are visible but not selectable
- Verify that the visual styling clearly indicates disabled times (grayed out)
- Ensure existing form validation still works correctly
- Test edge cases (no schedule, no appointments, full schedule)

## Dependencies
- No additional dependencies required
- Uses existing Bootstrap form-select styling which should automatically gray out disabled options
