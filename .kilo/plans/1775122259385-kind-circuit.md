# Plan: Fix Total Billed and Total Paid Cards to Show Full Dataset Values

## Problem
In the patient billing page (`/patient/billing`), the Total Billed and Total Paid cards currently show values based only on the current page of paginated billing records, rather than the entire dataset. When users navigate between pages, these values change incorrectly.

## Root Cause
In `PatientDashboardController.php::billing()` method:
1. Billing records are retrieved with pagination: `$billingRecords = Billing::where('patient_id', $user->id)->orderBy('invoice_date', 'desc')->paginate(10);`
2. Statistics are calculated from the paginated collection: `$totalBilled = $billingRecords->sum('total_amount');`
3. Since `paginate()` returns only the current page's items, the sums reflect only those items, not the full dataset.

## Solution
Separate the statistics calculation from the pagination:
1. Calculate totals using the full query (without pagination)
2. Get paginated records separately for display in the table
3. Pass both datasets to the view

## Implementation Steps
1. Modified `app/Http/Controllers/PatientDashboardController.php` in the `billing()` method:
    - Created a base query for billing records for statistics calculation
    - Calculated totals using `sum()` on the base query
    - Created a SEPARATE fresh query for pagination to avoid query builder state issues
    - Got paginated records using `paginate()` on the fresh query
    - Passed both the totals and paginated records to the view

2. No changes needed in the view file (`resources/views/patient/billing.blade.php`) as it already uses the correct variables.
